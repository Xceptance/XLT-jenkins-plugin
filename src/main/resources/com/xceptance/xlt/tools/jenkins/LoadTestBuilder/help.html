The following build variables will be made available during this step and can be used by subsequent actions.
  • XLT_RUN_FAILED: set to "true" if this step did not finished successfully
  • XLT_CONDITION_FAILED: set to "true" if a configured condition failed
  • XLT_CONDITION_CRITICAL: set to "true" if the configured number of past builds already failed by a condition (see advanced settings of "Validation")
  • XLT_CONDITION_ERROR: set to "true" if a condition could not be validated or caused an error
  • XLT_REPORT_URL: provides a direct link to the xlt report
  • XLT_CONDITION_MESSAGE: provides a preformatted message with informations about the failed conditions
